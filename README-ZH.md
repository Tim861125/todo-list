# Todo List - 每日待辦清單

一個使用 Vue 3 + TypeScript + Element Plus + Pinia 開發的現代化待辦事項管理應用。

## 技術棧

- **Vue 3** - 漸進式 JavaScript 框架
- **TypeScript** - JavaScript 的超集，提供型別安全
- **Vite** - 新一代前端建構工具
- **Pinia** - Vue 3 官方推薦的狀態管理方案
- **Element Plus** - 基於 Vue 3 的組件庫
- **Bun** - 快速的 JavaScript 執行環境和套件管理器

## 功能特性

### ✅ 核心功能
- **新增待辦事項** - 可設定標題、描述、優先級和截止日期
- **編輯待辦事項** - 修改已存在的待辦事項
- **刪除待辦事項** - 移除不需要的待辦事項
- **完成狀態切換** - 勾選/取消勾選待辦事項

### 🎯 進階功能
- **優先級管理** - 支援低、中、高三個優先級
- **截止日期** - 為待辦事項設定截止時間
- **日期提醒** - 自動標記今天、即將到期和已過期的待辦事項
- **分類篩選** - 可篩選全部、待完成、已完成、今日待辦
- **排序功能** - 依優先級或截止日期排序
- **批量操作** - 清除所有已完成的待辦事項

### 📊 統計功能
- 總計待辦數量
- 待完成數量
- 已完成數量
- 今日待辦數量

### 🎨 UI/UX 特性
- 響應式設計，支援手機和桌面
- 優雅的漸層背景
- 流暢的動畫效果
- 直觀的操作界面
- 待辦事項完成後的視覺反饋
- 日期狀態的顏色標示

## 專案結構

```
src/
├── components/
│   └── Todo/
│       ├── TodoApp.vue      # 主應用組件
│       ├── TodoForm.vue     # 新增/編輯表單
```

## 使用說明

### 新增待辦事項
1. 點擊右上角的「新增待辦」按鈕
2. 填寫待辦事項的資訊：
   - 標題（必填）
   - 描述（選填）
   - 優先級（必選：低/中/高）
   - 截止日期（選填）
3. 點擊「新增」按鈕

### 編輯待辦事項
1. 點擊待辦事項右側的編輯按鈕（藍色鉛筆圖標）
2. 修改相關資訊
3. 點擊「更新」按鈕

### 完成/取消完成待辦事項
- 點擊待辦事項左側的複選框即可切換完成狀態

### 刪除待辦事項
1. 點擊待辦事項右側的刪除按鈕（紅色垃圾桶圖標）
2. 在確認對話框中點擊「確定」

### 篩選待辦事項
使用篩選列的按鈕：
- **全部** - 顯示所有待辦事項
- **待完成** - 只顯示未完成的待辦事項
- **已完成** - 只顯示已完成的待辦事項
- **今日** - 只顯示今天截止的待辦事項

### 排序待辦事項
點擊「排序」下拉選單：
- **依優先級** - 高優先級排在前面
- **依截止日期** - 最早截止的排在前面

### 清除已完成項目
點擊「清除已完成」按鈕可批量刪除所有已完成的待辦事項。

## 待辦事項狀態說明

### 優先級
- 🔴 **高** - 重要且緊急的事項
- 🟡 **中** - 一般重要的事項
- 🔵 **低** - 不緊急的事項

### 截止日期狀態
- 🔴 **已過期** - 截止日期已過，紅色顯示
- 🟡 **今日** - 今天到期，橙色顯示
- 🔵 **即將到期** - 3天內到期，藍色顯示
- ⚪ **正常** - 其他日期，灰色顯示

## 資料儲存

目前版本使用 Pinia store 進行狀態管理，資料儲存在記憶體中。頁面重新整理後資料會清空。

## 開發指南

### 型別定義
所有的型別定義都在 `src/types/todo.ts` 中：
- `Todo` - 待辦事項的完整型別
- `TodoFormData` - 表單資料型別
- `TodoPriority` - 優先級列舉

### 狀態管理
使用 Pinia 的 Composition API 風格，store 位於 `src/stores/useTodoStore.ts`。

### 組件說明
- **TodoApp** - 整合所有子組件的主容器
- **TodoForm** - 使用 Element Plus 的對話框和表單元件
- **TodoItem** - 展示單個待辦事項的卡片
- **TodoList** - 使用 transition-group 提供動畫效果
